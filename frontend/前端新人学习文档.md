### 3.1 Vue 3 组合式 API（Composition API）

#### 为什么要有组合式 API？

Vue 2 用的是**选项式 API**（data、methods、computed 分开写），项目大了之后相关代码分散在各处。

Vue 3 的**组合式 API** 把同一个功能的代码放在一起，更易维护。

#### 对比一下

**选项式 API（Vue 2 风格）：**
```js
export default {
  data() {
    return { count: 0 }
  },
  computed: {
    doubled() { return this.count * 2 }
  },
  methods: {
    add() { this.count++ }
  }
}
```

**组合式 API（Vue 3 风格）：**
```js
import { ref, computed } from 'vue'

export default {
  setup() {
    const count = ref(0)            // 响应式变量
    const doubled = computed(() => count.value * 2)

    function add() {
      count.value++               // 修改 ref 要用 .value
    }

    return { count, doubled, add }  // 暴露给模板
  }
}
```

#### 本项目中实际例子（来自 `stores/user.ts`）：

```ts
import { ref, computed } from 'vue'

export const useUserStore = defineStore('user', () => {
  // ===== state（数据）=====
  const token = ref<string>(localStorage.getItem('token') || '')

  // ===== getters（计算属性）=====
  const isLoggedIn = computed(() => !!token.value)

  // ===== actions（操作函数）=====
  async function login(params) {
    const res = await loginApi(params)
    token.value = res.data.accessToken
    return res
  }

  function logout() {
    token.value = ''
  }

  return { token, isLoggedIn, login, logout }
})
```

#### 核心 API 速查

| API | 作用 | 示例 |
|-----|------|------|
| `ref()` | 创建响应式**变量** | `const count = ref(0)` → `count.value++` |
| `reactive()` | 创建响应式**对象** | `const obj = reactive({ name: '张三' })` → `obj.name = '李四'` |
| `computed()` | 计算属性 | `const doubled = computed(() => count.value * 2)` |
| `watch()` | 监听变化 | `watch(count, (newVal) => console.log(newVal))` |
| `onMounted()` | 生命周期-挂载后 | `onMounted(() => fetchData())` |
| `nextTick()` | 等 DOM 更新完再执行 | `nextTick(() => scrollToBottom())` |

### 3.2 TypeScript 类型系统

#### 为什么用它？

JavaScript 是弱类型语言，写错变量名不报错，上线后才发现 Bug。

TypeScript 提前帮你检查错误：

```ts
// ❌ JS：写错了也不报错
const user = { name: '张三' }
console.log(user.nmae)  // 没人报错，但运行结果是 undefined

// ✅ TS：写错直接报错
const user: { name: string } = { name: '张三' }
console.log(user.nmae)  // ❌ 编辑器直接标红：Property 'nmae' does not exist
```

#### 本项目中的类型定义（`types/user.ts`）：

```ts
export interface LoginParams {
  username: string
  password: string
}

export interface UserInfo {
  id: number
  username: string
  nickname: string
  avatar?: string       // ?: 可选属性，可以不存在
  role: 1 | 2            // 1=普通用户, 2=管理员
  menuPermissions?: string
}
```

#### 常见类型标注

| 写法 | 含义 |
|------|------|
| `let name: string` | 字符串 |
| `let age: number` | 数字 |
| `let ok: boolean` | 布尔 |
| `let arr: string[]` | 字符串数组 |
| `let obj: { id: number; name: string }` | 对象字面量 |
| `function fn(id: number): string { ... }` | 函数：参数 id 是数字，返回字符串 |
| `interface User { ... }` | 定义一个"用户"类型 |
| `type Status = 'pending' \| 'success'` | 联合类型，只能是这两个值 |

### 3.3 Pinia 状态管理

#### 什么是状态管理？

组件之间需要共享数据，比如：
- 用户登录了 → 所有页面都要知道"已登录"
- 购物车数量 → 商品页加一个，购物车页显示数量

Pinia 就是管理这些**全局共享数据**的地方。

#### Pinia 核心概念

```
Store（仓库）
    ├── State（数据）     → 相当于 Vue 2 的 data
    ├── Getters（计算）   → 相当于 computed
    └── Actions（操作）   → 相当于 methods
```

#### 本项目中实际例子（`stores/user.ts`）：

```ts
import { defineStore } from 'pinia'

export const useUserStore = defineStore('user', () => {
  // State：共享的数据
  const token = ref('')          // 登录令牌
  const userInfo = ref(null)    // 用户信息

  // Getter：基于 state 的计算值
  const isLoggedIn = computed(() => !!token.value)
  const isAdmin = computed(() => userInfo.value?.role === 2)

  // Action：操作数据的方法
  async function login(params) { ... }
  function logout() { ... }

  return { token, userInfo, isLoggedIn, isAdmin, login, logout }
})
```

#### 组件里怎么用？

```vue
<script setup>
import { useUserStore } from '@/stores/user'

const userStore = useUserStore()

// 模板里直接用
console.log(userStore.isLoggedIn)  // true/false
console.log(userStore.nickname)     // '张三'
</script>
```

### 3.4 Vue Router 路由

#### 什么是路由？

```
URL:  /spots           →  景点列表页面
URL:  /spots/1         →  景点详情页（id=1）
URL:  /hotels          →  酒店列表页
URL:  /admin/users     →  管理端用户管理页
```

路由就是 **URL → 页面组件** 的映射表。

#### 本项目路由配置（`router/index.ts`）：

```ts
const routes = [
  // 用户端：共用 UserLayout 布局
  {
    path: '/',
    component: () => import('@/layouts/UserLayout.vue'),
    redirect: '/home',                    // 访问 / 自动跳 /home
    children: [
      { path: '/home',      component: () => import('@/views/user/home/index.vue') },
      { path: '/spots',     component: () => import('@/views/user/spots/index.vue') },
      { path: '/spots/:id', component: () => import('@/views/user/spots/detail.vue') },
      { path: '/orders',    component: () => import('@/views/user/orders/index.vue'), meta: { requireAuth: true } },
    ]
  },
  // 管理端：共用 AdminLayout 布局
  {
    path: '/admin',
    component: () => import('@/layouts/AdminLayout.vue'),
    meta: { requireAuth: true, requireAdmin: true },  // 需要登录 + 管理员
    children: [ ... ]
  }
]
```

#### 路由守卫（路由拦截器）

每次跳转页面时，都会经过这个检查：

```ts
router.beforeEach(async (to, from, next) => {
  // 1. 设置页面标题
  document.title = `${to.meta.title} - 智慧旅游管理系统`

  // 2. 需要登录的页面
  if (to.meta.requireAuth) {
    const token = localStorage.getItem('token')
    if (!token) {
      return next({ name: 'Login', query: { redirect: to.fullPath } })
      // ↑ 踢到登录页，同时记住"登录后想去哪"
    }

    // 3. 管理端权限检查
    if (to.meta.requireAdmin && userStore.userInfo?.role !== 2) {
      return next('/')  // 非管理员踢回首页
    }
  }

  next()  // 通过检查，继续跳转
})
```

#### 路由传参的两种方式

**方式一：路径参数（动态路由）**
```ts
// 定义路由
{ path: '/spots/:id', component: SpotDetail }

// 跳转时
router.push('/spots/5')  // id=5

// 页面中获取
const route = useRoute()
route.params.id  // '5'
```

**方式二：Query 参数**
```ts
// 跳转时
router.push('/spots?id=5&tab=comment')

// 页面中获取
route.query.id    // '5'
route.query.tab   // 'comment'
```

### 3.5 Axios 请求封装

#### 什么是 Axios？

浏览器发 HTTP 请求的工具，类似于 `fetch`，但功能更强（拦截器、自动转换 JSON 等）。

#### 本项目的 Axios 封装（`api/axios.ts`）：

```ts
import axios from 'axios'

// 创建实例
const http = axios.create({
  baseURL: '/api',           // 所有请求的公共前缀
  timeout: 10000,            // 10秒超时
})

// 请求拦截器：每个请求发出去之前
http.interceptors.request.use(
  config => {
    const token = localStorage.getItem('token')
    if (token) {
      config.headers.Authorization = `Bearer ${token}`
      // ↑ 自动给每个请求带上 Token
    }
    return config
  },
  error => Promise.reject(error)
)

// 响应拦截器：每个响应回来之后
http.interceptors.response.use(
  response => {
    const res = response.data
    if (res.code !== 200) {
      // 业务错误（如 token 过期）
      return Promise.reject(res)
    }
    return res
  },
  error => {
    if (error.response?.status === 401) {
      localStorage.clear()
      window.location.href = '/login'
    }
    return Promise.reject(error)
  }
)

export default http
```

#### 封装后的接口文件（`api/spot.ts`）：

```ts
import http from './axios'

export function getSpotListApi(params) {
  return http.post('/v1/spot/list', params)
}

export function getSpotDetailApi(id) {
  return http.get(`/v1/spot/${id}`)
}
```

#### 组件里调用：

```vue
<script setup>
import { getSpotListApi } from '@/api/spot'

async function loadData() {
  const res = await getSpotListApi({ page: 1, pageSize: 10 })
  spots.value = res.data.records
}
</script>
```

### 3.6 Tailwind CSS + DaisyUI 样式

#### 什么是 Tailwind CSS？

传统 CSS：写一个 `.card { background: white; padding: 20px; border-radius: 8px; }`

Tailwind：直接在 HTML 上写类名
```html
<div class="bg-white p-5 rounded-lg shadow">
  内容
</div>
```

| 类名 | 含义 |
|------|------|
| `bg-white` | 白色背景 |
| `p-5` | padding 5 个单位（20px） |
| `rounded-lg` | 大圆角 |
| `shadow` | 阴影 |
| `text-center` | 文字居中 |
| `flex items-center` | Flexbox 居中 |
| `w-full` | width: 100% |
| `hover:bg-blue-500` | 鼠标悬停时变蓝色 |
| `dark:bg-gray-900` | 暗黑模式时背景 |

#### 什么是 DaisyUI？

在 Tailwind 基础上封装了一套好看的组件（按钮、卡片、弹窗等）：

```html
<!-- DaisyUI 按钮 -->
<button class="btn btn-primary">主要按钮</button>
<button class="btn btn-outline">描边按钮</button>

<!-- DaisyUI 卡片 -->
<div class="card bg-base-100 shadow-xl">
  <div class="card-body">
    <h2 class="card-title">标题</h2>
    <p>内容</p>
  </div>
</div>

<!-- DaisyUI 弹窗 -->
<dialog id="my_modal" class="modal">
  <div class="modal-box">
    <h3>弹窗内容</h3>
  </div>
  <form method="dialog" class="modal-backdrop"><button>close</button></form>
```

### 3.8 GSAP 动画

GSAP 是专业级动画引擎，做页面滚动动画用。

#### 常用场景

**场景一：入场动画（滚动到元素时触发）**

```ts
import gsap from 'gsap'
import { ScrollTrigger } from 'gsap/ScrollTrigger'
gsap.registerPlugin(ScrollTrigger)

// 当 .card 滚动到视口 80% 位置时，播放动画
gsap.fromTo('.card',
  { opacity: 0, y: 60 },    // 开始状态：透明、下移60px
  {
    opacity: 1, y: 0,        // 结束状态：显示、归位
    duration: 0.9,            // 动画时长 0.9 秒
    ease: 'power3.out',
    scrollTrigger: {
      trigger: '.card',
      start: 'top 80%',      // 元素顶部到视口80%时触发
      once: true              // 只触发一次
    }
  }
)
```

**场景二：文字逐词出现**

```ts
function splitTextByWord(el) {
  // 把文字拆成 <span>word</span> 逐词包裹
  return el.innerText.split(' ').map(w => `<span>${w}</span>`).join('')
}

gsap.fromTo(chars,
  { yPercent: 110, opacity: 0 },    // 从下方出来
  { yPercent: 0, opacity: 1, stagger: 0.05 }  // stagger=逐词延迟
)
```

### 3.9 高德地图 API

项目中用于景点和酒店的地图展示（`composables/useAmap.ts`）：

```ts
import AMapLoader from '@amap/amap-jsapi-loader'

export function useAmap(containerId: string, config: AmapConfig) {
  let map: any

  async function initMap() {
    // 加载高德地图
    const AMap = await AMapLoader.load({
      key: 'your-key',
      version: '2.0',
    })

    map = new AMap.Map(containerId, {
      zoom: config.zoom || 10,
      center: [config.longitude, config.latitude],
    })

    // 添加标记点
    config.markers.forEach(marker => {
      const m = new AMap.Marker({
        position: [marker.lng, marker.lat],
        title: marker.title,
      })
      map.add(m)
    })
  }

  return { initMap }
}
```

## 6. 典型开发流程

### 场景：新增一个"收藏景点"功能

#### Step 1：后端确认接口

先和后端约定好接口：
```
POST /api/v1/favorite/add     添加收藏
DELETE /api/v1/favorite/{id}  取消收藏
GET    /api/v1/favorite/list 我的收藏列表
```

#### Step 2：在 api/ 目录新增接口文件

```ts
// api/favorite.ts
import http from './axios'

export function addFavoriteApi(spotId) {
  return http.post('/v1/favorite/add', { spotId })
}

export function removeFavoriteApi(id) {
  return http.delete(`/v1/favorite/${id}`)
}

export function getFavoriteListApi(params) {
  return http.get('/v1/favorite/list', { params })
}
```

#### Step 3：在对应页面引入并使用

```vue
<script setup>
import { addFavoriteApi } from '@/api/favorite'

async function handleFavorite(spotId) {
  try {
    await addFavoriteApi(spotId)
    ElMessage.success('收藏成功')
    loadList()  // 刷新列表
  } catch {
    ElMessage.error('收藏失败')
  }
}
</script>

<template>
  <div v-for="spot in list" :key="spot.id">
    <h3>{{ spot.name }}</h3>
    <button @click="handleFavorite(spot.id)">❤️ 收藏</button>
  </div>
</template>
```

#### Step 4：如果需要全局状态（如收藏数量徽章）

在 `stores/` 新建 store：

```ts
// stores/favorite.ts
export const useFavoriteStore = defineStore('favorite', () => {
  const count = ref(0)

    const res = await getFavoriteCountApi()
    count.value = res.data
  }
```

## 附录：常见报错速查

| 报错信息 | 原因 | 解决 |
|---------|------|------|
| `Cannot read property 'xxx' of undefined` | 数据还没加载就用 | 加 `v-if="data"` 判空 |
| `Token expired` | Token 过期 | 后端返回 401，前端跳转登录 |
| `跨域 CORS 报错` | 后端没配置允许前端域名 | 后端加 CORS 配置 |
| `404 on module` | 依赖没装 | `pnpm install` |
| `Element Plus not registered` | 没全局注册组件 | `app.use(ElementPlus)` |
| `Pinia: "pinia" must be passed to "useXxxStore()"` | 没装 Pinia | `app.use(createPinia())` |
