## 一、项目概述

### 1.1 项目背景

随着旅游业蓬勃发展，游客对旅游体验的需求日益增长，传统旅游管理方式已无法满足个性化、智能化服务的要求。本系统旨在打造一个集景点展示、酒店预订、行程规划、互动社区于一体的**智慧旅游管理平台**。

### 1.2 项目目标

- ✅ 提供**沉浸式景点展示**（GSAP 动画 + 章节图文）
- ✅ 实现**智能化行程规划**与酒店预订
- ✅ 构建**实时互动社区**（游记分享、实时评论）
- ✅ 提供**数据可视化驾驶舱**（ECharts 大屏）
- ✅ 实现**菜单级权限管理**（管理员角色控制）
- ✅ 展示**高创意交互体验**（GSAP ScrollTrigger 动画）

### 1.3 预期成果

| 指标 | 目标 |
|------|------|
| 功能模块数 | ≥ 8 个 |
| 接口数量 | ≥ 40 个 |
| 前端页面数 | ≥ 25 个 |
| 创意动效页 | ≥ 5 个 |
| 代码规范 | TypeScript 严格模式 + ESLint |

### 3.2 核心业务流程

#### 管理员登录分流
```
访问 /login
    ↓
管理员 (role=2) → POST /api/v1/auth/login
    ↓
登录成功 → 跳转 /admin/dashboard（管理后台）
    ↓
普通用户 (role≠2) → 跳转 /home（用户前台）
```

#### 菜单权限控制
```
用户登录 → 获取 userInfo（含 menuPermissions 字段）
    ↓
menuPermissions = null → 拥有全部菜单权限
menuPermissions = "dashboard,spots,orders" → 仅显示指定菜单
    ↓
AdminLayout.vue visibleMenus computed 动态过滤侧边栏
```

#### 景点预订流程
```
用户浏览景点
    ↓
点击预订 → 选择日期 → 选择票型
    ↓
确认订单 → 登录/注册 → 支付
    ↓
支付成功 → 生成订单 → 发送通知
    ↓
入园游玩 → 评价打分
```

### 4.2 核心数据表（已实现）

#### 用户表 (users) ✅
```sql
CREATE TABLE users (
    id              BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '用户ID',
    username        VARCHAR(50) NOT NULL UNIQUE COMMENT '用户名',
    password        VARCHAR(255) NOT NULL COMMENT '密码(BCrypt加密)',
    nickname        VARCHAR(100) COMMENT '昵称',
    phone           VARCHAR(20) COMMENT '手机号',
    email           VARCHAR(100) COMMENT '邮箱',
    gender          TINYINT DEFAULT 0 COMMENT '0未知 1男 2女',
    role            TINYINT DEFAULT 1 COMMENT '1普通用户 2管理员',
    status          TINYINT DEFAULT 1 COMMENT '1正常 0封禁',
    menu_permissions TEXT COMMENT '菜单权限，逗号分隔菜单key（null=全部权限）',
    created_at      DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at      DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    deleted         TINYINT DEFAULT 0 COMMENT '逻辑删除',
    INDEX idx_role (role),
    INDEX idx_status (status)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='用户表';
```

**Menu Permissions 权限 key：**
- `dashboard` — 数据大屏
- `spots` — 景点管理
- `hotels` — 酒店管理
- `orders` — 订单管理
- `users` — 用户管理
- `articles` — 游记管理

## 五、API 接口设计

### 5.1 接口规范

```
Base URL: /api/v1
后端端口: 8080
前端端口: 5174
前端代理: /api/v1 → http://localhost:8080/api/v1

统一响应格式:
{
    "code": 200,        // 业务状态码
    "msg": "success",   // 消息
    "data": {},         // 数据体
    "timestamp": 1699999999999
}

状态码约定:
200 成功
400 参数错误
401 未登录
403 无权限
404 资源不存在
500 服务器异常
```

### 7.2 关键技术实现

**splitTextByWord 逐字拆分：**
```typescript
// 将标题元素按单词拆分为独立 span，GSAP 逐字动画
function splitTextByWord(el: HTMLElement): HTMLElement[] {
  const words = el.textContent?.split(' ') || []
  el.innerHTML = words
    .map(w => `<span class="word">${w}</span>`)
    .join(' ')
  return Array.from(el.querySelectorAll('.word'))
}
```

**clipPath reveal 图片动画：**
```typescript
gsap.fromTo(pics,
  { clipPath: 'inset(100% 0 0 0)', scale: 1.08 },
  {
    clipPath: 'inset(0% 0 0 0)', scale: 1,
    duration: 1.3, ease: 'power4.out',
    scrollTrigger: { trigger: item, start: 'top 78%', once: true }
  }
)
```

## 八、安全与权限设计（V1.1 完整实现）

### 8.1 JWT 认证流程

```
用户登录 → 验证密码 → 生成JWT令牌 → 返回token → 后续请求携带token
```

**核心流程：**
1. 前端调用 `/api/v1/auth/login`，携带用户名/密码
2. 后端验证成功后生成 JWT，设定过期时间（默认24小时）
3. 前端将 token 存入 localStorage
4. 后续所有请求在 Header 中携带 `Authorization: Bearer <token>`
5. Spring Security Filter 拦截并验证 token 有效性
6. 验证通过后将用户信息注入 SecurityContext，供后续业务使用

**Token 结构：**
```json
{
  "sub": "用户名",
  "userId": 123,
  "role": "ROLE_USER",
  "menuPermissions": ["spots", "hotels", "orders", "articles"],
  "exp": 1745184000
}
```

**安全措施：**
- ✅ Token 时效控制（24小时自动失效）
- ✅ 密码 BCrypt 加密存储（永不存明文）
- ✅ Spring Security 配置路径过滤（白名单）
- ✅ 菜单级权限控制（管理员 vs 普通用户）
- ✅ CORS 跨域限制
- ✅ 请求体大小限制（防止 DoS）

**技术实现：**
```typescript
// 消息结构
interface Message {
  role: 'user' | 'assistant'
  content: string
  loading?: boolean
}

// 发送消息 → 显示loading → 接收回复 → 渲染Markdown
async function sendMessage(text: string) {
  messages.value.push({ role: 'user', content: text })
  const aiMsg = { role: 'assistant', content: '', loading: true }
  messages.value.push(aiMsg)

  const res = await chatApi.sendMessage({
    message: text,
    history: getRecentHistory(3)  // 最近3轮
  })

  aiMsg.content = res.data.reply
  aiMsg.loading = false
}
```
